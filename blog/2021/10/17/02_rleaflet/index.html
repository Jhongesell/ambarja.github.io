Mapas interactivos en Leaflet desde R

Por Antony Barja | October 17, 2021

En las últimas décadas el avance tecnológico y la conexión a internet han tenido grandes impactos en el desarrollo de la evolución de la cartografía web, según Peng y Tsou en su libro Distributed Geographic Information Services for the Internet and Wireless Networks, el crecimiento del desarrollo del mapa web estuvo marcado por algunos contextos históricos, Tsou mencioná que el Tsunami de asia meridional y Huracán Katrina en el año 2005, alertaron a la comunidad mundial sobre la importancia de utilizar tecnología cartográfica web para gestionar y responder tales eventos, asimismo Tsou hace mención sobre 4 etapas importantes dentro del desarrollo del mapas web, la primera estuvo enfocada en la publicación de mapas estáticos en la web, donde el primer mapa web publicado fue PARC Map Viewer en el año 1993 (ver fig.01) (Veenendaal,Brovelli y Li, 2017).


El segundo enfoque estuvo orientado en el desarrollo del mapa web estático, la tercera en el mapa web interactivo y por último la distruibución de los servicios GIS, adicionado a este mismo enfoque Plewe (2007) hace mención a 5 etapas, donde el primer enfoque estuvo orientado a lo estático, la segunda a lo dinámico, la tercera a lo interactivo, el cuarto a los globos virtuales y finalmente el quinto a la computación en la nube y RIAs (desarrollo de widgets), existen diferentes conceptos relacionados al tema; sin embargo, lo que queda claro es que el desarrollo del mapa web creció gracias a la conexión a internet, y hoy en día existen muchísimos datos geolocalizados disponibles y capturados diariamente en línea y son usadas en aplicaciones web de uso diario, un claro ejemplo es Google Maps, Pokemon Go, instasights, entre otros.


Finalmente,tomando el enfoque de Tsou, otro contexto que quedará para la historía y que se evidenció la importancia de los mapas web o la geolocalización, la pudimos envidenciar durante el contexto de la pandemia, donde el trabajo realizado por la Universidad de John Hopkins mostró la evolución de los casos de coranovirus por país, este en su momento fue una herramienta de gran ayuda para la toma de desiciones, incluso algunas instituciones hoy en día siguen este modelo para implementar nuevas visualizaciones que muestren el avance de la cobertura de la vacunación contra el COVID-19, donde el núcleo de la visualización se enfoca en los mapas interactivos.


Si bien los primeros productos cartográficos en la web fueron herramientas simples que permitían de manera limitada a los usuarios la navegación, manejar el zoom, activación y desactivación de capas, hoy en día estos tienen una gama de funcionalidades incluido algunas funcionalidad de los SIG de escritorios, expandiendose más alla para incorporar requisitos de de aplicaciones cada vez mayor con la ventaja de compartir y analizar datos en la web.

Como puedes notar el empleo de los mapas web es muy importante dentro de la toma de decisiones, existen muchísimos softwares para el desarrollo de mapas webs entre ellas podemos mencionar dentro del frontend a lenguaje de HTML, CSS, o los lenguajes de programación como Java Script, Python o R, algunos frameworks de css como Bootstrap o Bulma,y dentro del backend tenemos a PostgreSQL, Django, Flask entre otros; sin embargo, en este pequeño post vamos a realizar un mapa web utilizando la librería leaflet, una librería de JavaScript especializada en el desarrollo de mapa web, pero dentro del software R con el paquete leaflet y derivados.

Requerimientos

Para este pequeño post vamos instalar los paquetes de:

  • tidyverse (paquete para ciencia de datos)
  • sf (paquete para manejar datos espaciales de tipo vector)
  • leaflet (paquete para generar mapas webs)
  • leaflet.extras (plugins para leaflet)
  • cptcity (paleta de colores).

1. Instalación de paquetes:

pkgs <- c(
  "tidyverse", "sf", "leaflet","htmltools"
  "htmlwidgets","leaflet.extras", "cptcity",)

install.packages(pkgs,dependencies = TRUE)

2. Llamado o activación de paquetes:

library(tidyverse)
library(leaflet)
library(leaflet.extras)
library(sf)
library(cptcity)
library(htmlwidgets)
library(htmltools)

3. Lectura de datos desde el portal de datos abiertos en un objeto sf

Para está ocación vamos a leer los datos disponibles del portal de datos abiertos del MINSA donde seleccionaremos los centros de vacunación disponibles para la provincia de Lima metropolitana.

vacunas_geo <- read_csv("https://www.datosabiertos.gob.pe/node/8298/download") %>% 
  st_as_sf(coords = c("longitud","latitud"),crs = 4326) %>% 
  mutate(id = row_number())

4. Lectura de los límites provinciales de la región de interés

lima_prov <- st_read("https://github.com/ambarja/gpkg-pe/raw/main/Lima_provincia.gpkg")

lista_id <- st_contains(lima_prov,vacunas_geo) %>% unlist()
vacunas_prov <- vacunas_geo %>% 
  filter(id %in% lista_id)

6. Para verificar los puntos dentro del área de interes, con la función plot() hacemos una visualización previa.

st_geometry(vacunas_prov) %>% plot()

7. Para visualizar todo los basemap disponibles en leafet usamos la siguiente función.

providers %>%

Observación: Una ventaja de leaflet es que nos ofrece una variedad de basemap y esto dependera de tus propia personalización y claro también de tu subscripción de algunos servicios de ciertas plataformas como HERE entre otros.

8. Para está ocasión emplearemos una basemap de ESRI de tipo satelital y otra de OpenStreetMap

logo <- tags$div(
   HTML(
     '<a href="https://ambarja.github.io/blog/2020/02/16/02_rleaflet/"> <img border="0" alt="ImageTitle" src="https://user-images.githubusercontent.com/23284899/137611650-26ca4b81-0f7b-4dea-afd2-21e1f44204f2.png" width="45" height="50"></a>')
 )

vacunas_prov %>% 
  leaflet() %>% 
  addProviderTiles(
    provider = "OpenStreetMap",
    group = "OSM") %>% 
  addProviderTiles(
    provider = "Esri.WorldImagery",
    group = "Satelital") %>% 
  addMarkers(
    clusterOptions = markerClusterOptions()
    ) %>% 
  addLayersControl(
    baseGroups = c("OSM","Satelital"),
    options = layersControlOptions(collapsed = FALSE)
    ) %>% 
  setView(lng =-77.02, lat =-12.03, zoom = 11) %>%
  addHeatmap(radius = 6) %>% 
  addSearchGoogle() %>% 
  addMiniMap(width = 120,height = 120) %>% 
  addControlGPS() %>% 
  addResetMapButton() %>% 
  addControl(logo, position = "bottomleft")


Finalmente, dependiendo del nivel de programación podrás personalizar a tu gusto y preferencia 😎, esto toma tiempo pero existe una amplia documentación desde lo más básico hasta un nivel avanzando.

Recuerda

La visualización interactiva de los mapas seguirá siendo un componente fuerte en la ciencia de datos espaciales, ya que el componente espacial y temporal es fundamental para la toma de desiciones y sobbre todo nos permite contextualizar eventos para tener una perspectiva de lo que viene y pude suceder en el territorio, y estos enfoques hoy en día tienen un mayor peso dentro de la neogeografía.
